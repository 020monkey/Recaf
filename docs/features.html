Specifics

Now here is a specific set of useful features. Not organized in any way (I need to get on that...)

  • Edit opcode-specific data
    • Editing existing opcodes shows a set of similar ones. For instance, editing an InvokeVirtual shows:
      • InvokeVirtual
      • InvokeSpecial
      • InvokeStatic
      • InvokeInterface
    • Other opcode-specific data is also presented. Values are updated instantaneously upon change.
  • Insert new opcodes
    • Opcodes are sorted into groups. Select the group related to the opcode you want to insert and a second dropdown containing the specific opcodes will appear. These groups all share the same values, so they will also appear below the opcode dropdown.
  • Save and load blocks of opcodes
    • Save existing code for use later.
    • Insert saved blocks into programs.
  • Display local variable
    • When editing variable-related opcodes data from the local variable table (If available) is displayed for convienence.
  • Syntax highlighting
    • Opcodes are bolded and additional information is color coded. You are able to change the colors through rcstyle.json
  • Search for declarations/references of/to strings, fields, methods, and classes
    • When given the option, searches are case insensitive by default. Searching for "aa" will show results for "AA" and "aAaA" for instance.
    • Searching for references does not require all fields to be entered. This allows you to search for all references to a class, or all references to any member of a given name or type.
  • Inline jump logic hints
    • Next to all jump opcodes is a representation of their logic. These can be disabled in the options menu/file.
  • Label editing
    • Labels are a part of a few opcode groups and method attributes (jumps, switches, try-catch). Labels in a method are given arbitrary names based on their position.
  • Control flow highlighting
    • Control flow (Jumps and switches) have their destinations color coded. For jumps, green is the destination of a successful jump. Red for failure. For switches, yellow are the potential destinations, red is the default destination. You can also select a label to check where it is referenced.
    • Examples:
      • Goto
      • Conditional Jumps
      • Tableswitch / Lookupswitch
      • Label references
  • Decompile with CFR
    • Whole classes
    • Single methods
    • Recompile changes made to decompiled code
  • Automate tasks with plugin API
  • Attach to running java processes
    • Example editing Minecraft
