Updating and compiling with Java 9+

Why?

Seeing that Java 8 is still very popular and lots of people have not yet moved to later versions (Let alone know that new versions exist) the builds that are provided on github will be based on Java 8. This short guide will allow you to compile Recaf to work on later versions of Java.

Updating the pom.xml

  1. Update the maven-compiler-plugin plugin, replace <source>1.8</source> and <target>1.8</target> with <release>$VERSION</release> where $VERSION is the integer release of the JDK, such as 9, 10, 11, 12, etc.
  2. Update the controlsfx dependency in the pom.xml such that the version is changed to the match the version of Java you want to run on. If you're not sure which version to use, use the most recent version.
  3. Adding the OpenJFX module (Required for Java 11+)
    • Add the JavaFX web dependency that matches the version of Java you want to run on. It will automatically resolve the rest of the OpenJFX dependencies required to compile.

Compiling

You can then build from the source code with maven (Open a terminal and enter mvn package assuming you have Maven installed).
