The Assembler

WIP/TODO: Article will be updated shortly to include the information below:

  • Controls
    • Access: Method access
    • Name: Method name
    • Desc: Method descriptor, shows parameters and return type
    • Verify: Flag for enabling the assembler's verification step
    • Local vars: Flag for enabling local variable data being emitted by the assembler
  • Code-area
    • Errors: When a line cannot be parsed by the assembler, it will be marked red. Hovering over the red dot will give the reason why the parse failed. Some potential reasons include:
      • Incorrect variable name/index specified
      • Incorrect label name specified
      • Invalid instruction format (Missing required parameter / invalid parameter value specified)
      • Invalid opcode specified
    • Instruction formats (TODO: Table)
    • Tab-completion
    • Named variable syntax
      • N literal variable index
      • this maps to index 0
      • pN maps to N'th method parameter. Starts at N=1. (Based off bytecode variable index, so double/longs take two indices... an example will be included with more detail)
      • anythingElse local variable auto-assigned index via post-process in assembler
  • Limitations
    • No support for try-catch blocks, they will have to be added after compilation
